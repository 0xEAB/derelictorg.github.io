DerelictSDL2

DerelictSDL2 is a dynamic D binding to the 2.x series of the Simple DirectMedial Layer (SDL) library. DerelictSDL2 2.0.0 and higher supports all releases of SDL2 from 2.0.0 and up via Derelict's SharedLibVersion feature. Older versions of DerelictSDL2 are tied to specific versions of SDL2. New projects using DerelictSDL2 should use 2.0.0 or higher.

DerelictSDL2 also includes bindings to the following SDL satellite libraries:

  • SDL_image
  • SDL_mixer
  • SDL_net
  • SDL_ttf

The following sections explain how to build and use DerelictSDL2. See the page Using Derelict for more information.

Using With DUB

The easiest way to use DerelictSDL2 is to use DUB to manage your project and add it as a dependency in your project configuration file. Those using the JSON format should add the following to the "dependencies" field of dub.json.

            "derelict-sdl2": "~>2.0.0"

Those using the Simple Declarative Language (SDLang) should add the following to dub.sdl.

            dependency "derelict-sdl2" version="~>2.0.0"

By using the format ~>2.0.0, the version of DerelictSDL2 will be constrained between 2.0.0 and 2.1.0. This allows you to get the latest version when compiling a project for the first time or when running dub upgrade in your project directory.

Using Without DUB

It is not necessary to use DUB to manage your project in order to use DerelictSDL2, but it is still required to build it. To do so, make sure DUB, DMD and git are all available on your system path, then execute the following commands to obtain the source code.

            git clone https://github.com/DerelictOrg/DerelictSDL2.git
            cd DerelictSDL2
            git checkout 2.0.0

The master branch is the default branch and may contain changes that have not yet been pushed the 2.0.0 branch. In order to make sure that you're using the latest release, checkout the 2.0.0 branch. The following will build the library.

            dub build -brelease -ax86_64

Drop the -b argument on the DUB command line for a debug build and the -a argument for 32-bit. The library will be created in DerelictSDL2/lib. DerelictSDL2 depends on DerelictUtil. DUB will pull it down and install it in the DUB cache. However, you will still need to build it and link it into your application. You can do that by building the version in the DUB cache or cloning the DerelictUtil repository seperately and building that.

Once you have both DerelictSDL2 and DerelictUtil compiled, you will need to link both libraries into your application. The easiest thing to do will be to copy them to a common directory where your compiler can find them, either through the system path or with a command line switch, or add them to your project settings in your IDE.

Loading

To load the SDL2 library with DerelictSDL2, call one of the overloads of the load method on the DerelictSDL2 instance.

            DerelictSDL2.load();

This no-argument version will attempt to load the SDL2 library using a default set of system-specific library names to look for on the system search path. Overloads accept one of more custom library names to search for. DerelictSDL2 also supports Derelict's SharedLibVersion feature. For example:

            DerelictSDL2.load(SharedLibVersion(2, 0, 2));

This requires SDL 2.0.2 as a minimum and will fail if only SDL 2.0.1 or lower is available, but will successfully load if SDL 2.0.2 or higher is available. See the loading documentation for more information relevant to all shared libraries.
